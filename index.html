Sistemas Distribuídos e Webservices

1. O que é um Webservice?

Webservice é um serviço disponível na internet que permite que sistemas diferentes se comuniquem automaticamente. Ele funciona como uma “ponte” para troca de dados entre sistemas, aplicativos, servidores e dispositivos.

Exemplo simples:

• Seu site envia um CEP • O Webservice (API) responde com: rua, bairro, cidade, estado

Por que usar webservices?

  • Integra sistemas diferentes
  • Automatiza processos
  • Evita trabalho manual
  • Conecta empresas
  • Permite criar aplicativos modernos

2. Tipos de Webservices

✔ REST (o mais usado hoje)

• Usa URLs simples • Usa JSON • Rápido, leve e moderno • Exemplo: https://viacep.com.br/ws/01001000/json/

✔ SOAP (antigo, ainda usado em bancos)

• Usa XML • Muito seguro • Mais pesado • Exige contratos (WSDL)

✔ APIs

API é um conjunto de regras que um sistema disponibiliza para outros consumirem informações. Webservice é um tipo de API disponível via internet.

3. Formatos de Dados

✔ JSON (mais usado)

{
  "cep": "01001-000",
  "logradouro": "Praça da Sé",
  "bairro": "Sé",
  "localidade": "São Paulo"
}

✔ XML

<endereco>
    <cep>01001-000</cep>
    <logradouro>Praça da Sé</logradouro>
</endereco>

4. Como funciona a comunicação?

Cliente → Servidor

1. O cliente (seu site/app) envia uma requisição.

Servidor → Cliente

2. O Webservice responde com dados.

Métodos HTTP mais usados:

  • GET – Buscar informações
  • POST – Enviar informações
  • PUT – Atualizar informações
  • DELETE – Remover informações

Status HTTP

  • 200 – OK
  • 400 – Requisição errada
  • 401 – Não autorizado
  • 404 – Não encontrado
  • 500 – Erro no servidor

5. Segurança em Webservices

• Token • API Key • OAuth 2.0 • HTTPS • Controle de IPs

Exemplo de Header com Token:

Authorization: Bearer 9a8sd7a90sd8a09s8d...
